FÁZE 7 – FE ROUTING INCIDENT (09.01.2026)

Projevil se problém: klik na Create vedl na /create → FORPSI vracel 404, případně /#/create renderovalo Homepage.

Příčina: App layout renderoval pouze Homepage bez Routes (Create route nebyla aktivní) a zároveň existovaly hardlink odkazy href="/create", které obcházely HashRouter.

Náprava: ukotveno pravidlo “FORPSI = HashRouter only”, zákaz /create hardlinků, přidán regresní sanity check do FLOW a LOCK položka do BOARD.
[FIX] Header nav styles + routing /create
Problém nebyl v kódu, ale v nasazeném buildu a cache CSS.
Řešení: sjednocení .nav-link (buttony), CTA logika + cache-busting kontrola.

🎧 FÁZE 13 – SHAREPAGE PLAYER UI + RETRY FIX (ERR_CONNECTION_RESET) (2026-01-31)
--------------------------------------------------

Důvod:
- SharePage používala nativní <audio controls> (šedý browser UI mimo Melody4U styl).
- Při kliknutí na „Přehrát“ docházelo k situaci:
  - /consume vrátil media/ticket URL
  - request na mp3 selhal (Chrome Network: ERR_CONNECTION_RESET, často po 302 redirectu na presigned URL)
  - FE vyhodnotil chybu jako „konec“ a hned načetl stav → UI ukázalo „vyčerpáno“
  - uživatelsky to působilo jako okamžité spálení přehrání bez přehrávání.

Změny (FE):
- SharePage přepracována na custom Melody4U audio player (sjednocený styl s Create).
- Přidán režim „Retry“ bez dalšího /consume:
  - po /consume se uloží ticketUrl
  - při chybě přehrávání se NENAČÍTÁ hned stav (neoznačuje se jako vyčerpáno)
  - tlačítko se změní na „Zkusit znovu“ a znovu spustí přehrávání ze stejného ticketUrl
- Stav (remaining/locked) se načítá až při reálném dokončení přehrávání (onEnded),
  aby se zabránilo falešnému „vyčerpáno“ po síťové chybě.

Poznámka:
- Samotná síťová příčina (ERR_CONNECTION_RESET na presigned mp3) může být mimo FE (storage/redirect/headers).
- FE fix je ochrana UX a prevence falešných stavů. CORE logika zůstává beze změny.

Výsledek:
- SharePage vypadá konzistentně se zbytkem UI (bez šedých browser prvků).
- Při chybě streamu už uživatel nevidí okamžitě „vyčerpáno“ a může retry bez dalšího odečtu.
- Ověřeno: po úpravě přehrávání funguje.

📩 FÁZE 14 – CREATE “HOTOVO” = SDÍLENÍ (NE PLAYBACK) + ČASOVÝ NÁHLED ODESÍLATELE (2026-01-31)
--------------------------------------------------

Důvod:
- Po vytvoření mixu (Create flow) se aplikace přepínala rovnou na obrazovku přehrání (SharePage).
- To je špatně z pohledu produktu:
  - po mixu chceme primárně “odeslat dárek”
  - přehrání má být až pro PŘÍJEMCE po otevření odkazu
- Odesílatel ale potřebuje možnost zkontrolovat, že přání je “podle představ” před odesláním.

Rozhodnutí:
1) Po úspěšném mixu a vytvoření shareId se zobrazí SHARE SCREEN (odesílatel).
   - žádné automatické přesměrování na přehrávání
   - CTA jsou sdílecí možnosti

2) Share screen obsahuje sdílecí kanály:
   - kopírovat odkaz
   - WhatsApp
   - Facebook Messenger
   - e-mail (mailto)
   - (volitelně) QR kód
   - (volitelně) “zkopírovat text zprávy” s odkazem

3) Náhled pro odesílatele:
   - časově omezený preview (např. 1 hodina)
   - preview nesmí odečítat přehrání z play-limit
   - preview je ochranný UX režim, aby se neobcházela hodnota tarifu

Implementační poznámka:
- Preview bude řešen tokenem (previewToken) s expirací 1 hodina
- Token typ PREVIEW:
  - neodečítá used_plays
  - může být vázaný na IP/UA (minimalizace sdílení)
- Playback pro příjemce zůstává přes /p/:id + /consume (STABLE)

Výsledek:
- UX po vytvoření přání odpovídá “teď to pošli”
- příjemce uvidí playback až po otevření odkazu
- odesílatel má bezpečný náhled bez “spálení” přehrání

Doplnění (ověřeno v praxi):
- Share screen (odesílatel) funguje a je použitelný jako produkční UI.
- Sdílení je “mobile-aware”:
  - mobil: otevírá aplikace (WhatsApp/Messenger/Telegram), e-mail přes mail app
  - desktop: WhatsApp Web, Telegram web, e-mail volba Gmail/Outlook/Jiný
- Uhlazené UI:
  - odstraněny debug informace (sender link, share ID)
  - odstraněny pracovní prvky (“Sdílet…”, “kopírovat odkaz”, “kopírovat text + odkaz”)
  - přidán textbox pro průvodní zprávu, odkaz se přidává automaticky na konec
